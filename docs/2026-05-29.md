# 2026-05-29

## PC Trails admin — edit/delete fixed

**Reported:** the PC "Trails" editor said "Saved" but edits and deletes did nothing.

Four things were wrong; all had to be addressed:

1. **Empty table** — `public.trails` was never seeded, so [[trail_service.dart]] silently served the read-only bundled JSON. Edits/deletes hit nonexistent DB rows. *Fixed by running PC → "Seed from bundle".*
2. **Split-brain admin** — `bremnermail@gmail.com` had `profiles.is_admin=true` (so the tab showed) but no [[admin_users]] row, so [[is_admin]]() was false and RLS filtered every write to 0 rows. *Fixed by adding the admin_users row; verified `is_admin()=true`.*
3. **Silent success** — [[trail_repository.dart]] didn't check affected rows, so a blocked write still reported "Saved". *Now `.select()`s the affected rows.*
4. **Recompute** — [[Trail Model]]`.fromJson` always recomputed `difficulty` + `elevation_gain_m`, discarding admin edits. *Now stored values win; added a `published` field.*

## Catalogue dedup

- The bundle seeds ~29 routes **twice** (hyphen-id + underscore-id twins, e.g. `hc_to_caracal_cave` / `hc-to-caracal-cave`). Deleting one left the twin → "the route won't go away."
